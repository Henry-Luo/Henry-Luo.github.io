使用 TypeHandler 来转换你的 Java 对象

有时，我们有这样的场景或需求。例如某种平台上有各种各样的申请，如果你给每一种申请单独维护一张表，虽然看似简单，但是管理起来数量多也麻烦。如果以后有新的申请又得新建表，涉及到数据库的操作，几乎所有的公司都有一套复杂且严格的流程或手续，所以拓展起来也不是那么方便了。

于是，自然想到维护一张统一的表，该表其中一个字段 (varchar) 对应各种不同申请。这就引出了这样一个问题，“如何在不同的 Java 对象(每种申请对应一个 Java 对象)，与数据库的 varchar 之间互相转换呢？”

这里，我们使用 Mybatis 的 TypeHandler 来解决这个问题。

其中的 apply_content 字段就对应各种申请，需要指定 TypeHandler 来转换。那么 JsonTypeHandler 又是什么样的呢？

可以看到，使用 Json (Json 序列化可以选择 FastJson 等)来序列化成字符串，往数据库里存，对应 varchar。取出来的时候，在 Handler 内部处理逻辑中，根据 type 拿到类型，再转换成了相应的对象。这里有点遗憾，需要自己根据类型再判断转换，如果能自己找到类型进行转换就完美了。
