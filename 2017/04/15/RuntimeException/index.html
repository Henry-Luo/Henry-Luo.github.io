RuntimeException

写这篇文章之前，还是感到很惭愧的。毕竟没什么难度，只是因为自己之前的基础不够扎实，不了解这个知识点罢了。所以还是要记录一下，加强记忆好好长长记性，鞭策自己。

起因是阅读一个第三方的 SDK 代码，来寻找其中的潜在 BUG。发现有一段代码抛出了异常，却没有使用 try/catch 捕获，也没有直接再抛出，感到有点疑惑。后来查阅了相关资料才发现，代码里面的自定义异常是继承的 RuntimeException，RuntimeException 表示运行时异常，是一种 unchecked exception，可以不捕获这种异常。通常也不建议捕获 RuntimeException 异常，因为它代表了一种意料之外的、本不该发生的异常，代表程序中有 BUG，catch RuntimeException 的意义不大，而应该去修复它。相比于我们经常需要 try/catch 的 checked exception，这是主要的不同之处。

好了，现在清楚了。另外最开始提到的 BUG 也找到了，使用 HTTP 连接池一定要注意连接泄露，这个以后有空再写啦。
